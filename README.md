## 1. Project Overview

**Dayflow** is a role-based Human Resource Management System (HRMS) designed to digitize and streamline core HR operations such as employee onboarding, authentication, attendance tracking, leave (time-off) management, employee profile management, and salary visibility.

The system is built with a **clear separation between Admin and Employee roles**, ensuring security, data privacy, and real-world HR workflow simulation.

This project is developed as a **hackathon-ready working MVP**, with scope for future enterprise-level extensions.

---

## 2. Problem Statement

Most small to mid-scale organizations still rely on:

* Manual attendance tracking
* Email / WhatsApp based leave approvals
* Scattered employee data
* Poor visibility into employee work status

This results in:

* Data inconsistency
* Human errors
* Lack of transparency
* Poor HR decision-making

**Dayflow solves this by providing a centralized, role-based HRMS platform.**

## 4. Authentication & Login Module

### 4.1 Why No Public Sign-Up?

* Employees **cannot register themselves**
* Only **Admin creates employee accounts**
* This prevents:

  * Fake accounts
  * Unauthorized access
  * Role manipulation

---

### 4.2 Login Page (Mandatory)

**Fields:**

* Email ID / Login ID
* Password

**Flow:**

1. User enters credentials
2. System validates credentials
3. Role is identified (Admin / Employee)
4. User is redirected to respective dashboard

**Edge Cases:**

* Invalid email ‚Üí error message
* Wrong password ‚Üí error message
* Disabled account ‚Üí access denied
* First-time login ‚Üí force password change

---

### 4.3 Password Handling

* Password is **auto-generated by system**
* Sent to employee via **email**
* On first login:

  * User **must change password**
* Password rules:

  * Minimum length
  * Alphanumeric
  * Secure hashing (backend)

### 6.5 Time Off (Leave) Tab ‚Äì Employee

#### 6.5.1 Leave Overview

* Shows:

  * Start date
  * End date
  * Leave type
  * Status (Pending / Approved / Rejected)

---

#### 6.5.2 New Leave Request

**Fields:**

* Employee Name (auto-filled)
* Leave Type:

  * Paid Leave
  * Sick Leave
* Validity Period (Start ‚Üí End)
* Allocation (number of days)
* Attachment (mandatory for sick leave)
* Buttons:

  * Submit
  * Discard

**Validations & Edge Cases:**

* Cannot request leave beyond available balance
* Overlapping leave dates not allowed
* Sick leave requires document upload
* Past date leave not allowed
* On submit ‚Üí status = Pending
